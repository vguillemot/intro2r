Loprateur : est trs utilis en R. Sa syntaxe est la suivante

  • \(i\):\(j\) va crer une suite dentiers de \(i\) \(j\). Les entiers peuvent tre ngatifs ou positifs, et on peut peut avoir \(i < j\) ou \(i > j\), ou mme \(i = j\).

Attention bien mettre des parenthses dans le cas dentiers ngatifs!

Les noms dobjets

Rgles absolues :

  • Commence par une lettre ou un point, si le premier caractre est un point, le deuxime ne peut pas tre un chiffre,
  • Pas despace
  • Pas de caractres correspondant des oprations (+, -, *, /, ^, **, etc.)
  • Les minuscules et les majuscules sont diffrentes !
  • Certains “mots-clefs” sont strictement interdits (NA, TRUE, FALSE, for, if, else etc.)
  • MAIS on peut utiliser un nom dobjet qui existe dj !

Bonnes pratiques :

  • Utiliser un nom qui a du sens
  • Ne pas utiliser des noms dobjets qui existent dj et que lon ne souhaite pas craser !

Loprateur de slection classique : [ , ]

  • Pour slectionner les colonnes 3 5 : fruits[, 3:5]
  • Pour enlever les trois premiers fruits : fruits[-(1:3), ]
  • Pour enlever les cinq premires variables : fruits[, -(1:5)]

Extraction avec des Boolens

Comment extraire les fruits * dont la teneur en eau est suprieure 60 ? * exotiques ? * secs contenant moins de 40g/100g de sucres ?

Rponse : en utilisant des vecteurs boolens

  1. Crer le vecteur de boolens fruits$Eau >= 60
  2. Utiliser le rsultat dans les crochets carrs fruits[fruits$Eau >= 60, ]

Ne pas oublier la virgule !

Le principe

Pour un vecteur v :

  • v[bool] extrait les valeurs de v pour lesquelles bool est vrai (TRUE). Contrainte : v et bool doivent contenir le mme nombre dlments.

Pour un tableau tab :

  • tab[brow, ] pour extraire les lignes
  • tab[, bcol]
  • Contrainte 1 : brow doit avoir autant dlments que tab de lignes
  • Contrainte 2 : bcol doit avoir autant dlments que tab de colonnes

Attention

Vous verrez souvent des oprations logiques lintrieur des crochets carrs : cela permet daller plus vite !

Par exemple, en deux tapes :

  1. bool <- fruits$groupe == "secs" & fruits$Sucres < 40
  2. fruits[bool, ]

Devient, en une tape :

  • fruits[fruits$groupe == "secs" & fruits$Sucres < 40, ]

Attention bis

On peut combiner deux mthodes dextraction de donnes pour un tableau : une sur les lignes et une sur les colonnes !

Par exemple : tab[brow, icol], o brow est un vecteur de boolens et icol un vecteur dindices.

On utilise :

  • rownames(tab) pour connatre le nom des lignes
  • colnames(tab) pour connatre le nom des colonnes

Et, en bonus, on peut :

  • changer les noms des lignes rownames(tab) <- new1
  • changer les noms des colonnes colnames(tab) <- new2

Et, en super bonus, on peut :

  • modifier quelques noms de lignes rownames(tab)[sel1] <- new1
  • modifier quelques noms de colonnes rownames(tab)[sel1] <- new2

Rappel : la covariance

Permet de mesurer le degr de co-variation de deux variables :

\[ \operatorname{cov}(x, y)=\frac{1}{n-1} \sum_{i=1}^{n}\left(x_{i}-m_{x}\right)\left(y_{i}-m_{y}\right) \]

Rappel : corrlation de Pearson

Cest une covariance normalise entre -1 et 1 !

\[ \operatorname{cor}(x, y)=\frac{\operatorname{cov}(x, y)}{\sqrt{\operatorname{var}(x)} \sqrt{\operatorname{var}(y)}} \]

Rappel : corrlation de Spearman

Cest la corrlation (de Pearson) calcule sur les rangs !

\[ \rho = \operatorname{cor}(r_x, r_y), \] avec \(r_x\) le vecteur des rangs de \(x\) (rank(x)), et \(r_y\) le vecteur des rangs de \(y\) (rank(y)).

Rappel (?) : corrlation de Kendall

  • Paire concordante : \(\left(x_{i}<x_{j} \text { et } y_{i}<y_{j}\right)\) OU \(\left(x_{i}>x_{j} \text { et } y_{i}>y_{j}\right)\)
  • Paire discordante : \(\left(x_{i}<x_{j} \text { et } y_{i}>y_{j}\right)\) OU \(\left(x_{i}>x_{j} \text { et } y_{i}<y_{j}\right)\)

\[ \tau = \displaystyle \frac{n_C - n_D}{n_0}, \] avec \(n_C\) le nombre de paires concordantes, \(n_D\) le nombre de paires discordantes et \(n_0\) le nombre total de paires de points.
